# 시장 저점 판독 텔레그램 봇 PRD

## 1. 문서 목적
이 문서는 `01_domain.md`에서 정의한 문제와 범위를 바탕으로, 제품이 이 문제를 어떻게 풀고 언제 사용되는지 구체화한다.

핵심 목적은 장 마감 후 입력된 시장 데이터를 기준으로 저점 상태를 판정하고, 그 결과를 텔레그램 리포트로 전달하는 최소 기능 제품을 정의하는 것이다.

## 2. 사용 시점
- 사용 시점은 한국 주식시장 장 마감 이후다.
- 사용자는 마감 데이터를 확보한 뒤 당일 시장 상태를 점검하기 위해 이 제품을 사용한다.
- 자동 수집이 가능하더라도, 초기 버전에서는 수동 입력으로도 동일한 흐름을 수행할 수 있어야 한다.

## 3. 제품 목표
- 사용자가 장 마감 후 시장 상태를 1분 이내에 점검할 수 있어야 한다.
- 동일한 데이터가 들어오면 동일한 판정 결과가 나오도록 일관된 규칙 기반 판정이 가능해야 한다.
- 결과는 숫자와 해석이 함께 포함된 한국어 리포트로 제공되어야 한다.

## 4. 사용자 시나리오
- 상세 사용자 흐름은 `03_user_scenario.md`를 따른다.

## 5. 제품이 문제를 푸는 방식
- 장 마감 기준 시장 데이터를 입력받는다.
- 입력값을 정해진 스키마와 허용값 기준으로 검증한다.
- 검증된 데이터를 규칙 기반 저점 판정 로직에 넣는다.
- 판정 결과와 근거를 한국어 리포트로 생성한다.
- 생성된 리포트를 텔레그램으로 전달한다.
- 자동 수집 실패 시에도 수동 입력 경로로 동일한 판정 흐름을 유지한다.

## 6. 핵심 기능 요구사항
### 6.1 데이터 입력
- 시스템은 장 마감 기준 데이터를 입력받아야 한다.
- 초기 버전에서는 수동 입력을 우선 지원해야 한다.
- 입력 데이터는 사전에 정의된 스키마를 따라야 한다.
- 추후 자동 수집 방식으로 확장 가능해야 한다.

### 6.2 데이터 검증
- 필수 필드 누락 여부를 검사해야 한다.
- 허용값이 정해진 필드는 사전 정의된 값만 허용해야 한다.
- 입력 형식 오류가 있으면 판정을 진행하지 않고 오류를 반환해야 한다.

### 6.3 저점 판정
- 판정은 문서화된 규칙 기반 로직으로 수행해야 한다.
- 결과는 반드시 아래 3단계 중 하나여야 한다.
- `추가 조정 필요`
- `저점 근접`
- `진 바닥 확인`
- 판정 결과에는 근거가 되는 주요 신호와 점수 또는 조건 반영 결과가 포함되어야 한다.

### 6.4 리포트 생성
- 출력은 한국어여야 한다.
- 리포트는 아래 항목을 포함해야 한다.
- `현 시점 저점 판독 결과`
- `판단 근거 (데이터 분석)`
- `매크로 상황 분석`
- `최종 투자 의견`
- `주목할 업종`
- 숫자 데이터와 해석 의견은 분리해서 작성해야 한다.
- 투자 자문처럼 단정적으로 표현하지 않아야 한다.

### 6.5 텔레그램 전송
- 리포트를 텔레그램 메시지 형식으로 전송할 수 있어야 한다.
- 전송 실패 시 재시도 또는 실패 로그 확인이 가능해야 한다.

## 7. 비기능 요구사항
- Python 3.11 환경을 기준으로 동작해야 한다.
- 환경변수와 비밀값은 `.env`로 분리해야 한다.
- 테스트 코드가 포함되어야 한다.
- 로직은 재현 가능하고 결정적이어야 한다.
- 장애 상황에서도 수동 입력 기반 경로는 유지되어야 한다.
- 한국어 출력 품질은 간결하고 전문적인 수준을 유지해야 한다.

## 8. 초기 버전 범위
### 포함
- 수동 입력
- 규칙 기반 판정
- 근거 포함 리포트 생성
- 텔레그램 전송

### 제외
- 종목별 매수/매도 추천
- 실시간 장중 분석
- 뉴스 감성 분석
- 머신러닝 기반 예측

## 9. 성공 기준
- 사용자가 장 마감 데이터를 입력했을 때 판정 결과가 정상 생성된다.
- 같은 입력값으로 반복 실행 시 동일한 결과가 생성된다.
- 텔레그램 메시지가 정해진 형식으로 전송된다.
- 누락 또는 잘못된 입력값에 대해 시스템이 명확한 오류를 반환한다.
