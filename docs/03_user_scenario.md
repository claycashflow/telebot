# 사용자 시나리오

## 1. 문서 목적
이 문서는 `01_domain.md`와 `02_prd_check_bottom_bot.md`를 바탕으로, 제품 요구사항을 실제 사용 흐름과 예외 흐름으로 보조 설명한다.

목적은 기능 목록을 반복하는 것이 아니라, 실제 사용 장면에서 입력, 판정, 리포트 생성, 전달이 어떤 순서로 일어나는지 구체적으로 보여주는 것이다.

## 2. 문서 성격
- 이 문서는 PRD의 지원 문서다.
- 제품이 어떤 흐름으로 사용되는지 예시 중심으로 설명한다.
- 기능 요구사항의 기준 문서는 아니며, 구현 세부 규칙은 PRD와 로직 문서를 따른다.

## 3. 핵심 사용자
- 한국 주식시장 마감 후 시장 상태를 빠르게 점검하려는 개인 투자자
- 감정이 아니라 규칙 기반으로 시장을 해석하고 싶은 사용자
- 매일 같은 형식의 저점 판독 리포트를 받고 싶은 사용자

## 4. 대표 시나리오
### 시나리오 A. 장 마감 후 수동 입력으로 시장 상태 점검
- 사용자는 장 마감 후 KOSPI/KOSDAQ 종가와 주요 보조지표를 준비한다.
- 사용자는 봇 또는 입력 인터페이스를 통해 데이터를 입력한다.
- 시스템은 입력값이 스키마에 맞는지 검증한다.
- 검증이 통과되면 규칙 기반 로직으로 시장 저점 상태를 판정한다.
- 시스템은 판정 결과와 근거를 한국어 리포트로 생성한다.
- 최종 리포트는 텔레그램으로 전송된다.

### 시나리오 B. 자동 수집 실패 후 수동 전환
- 자동 데이터 수집이 실패하거나 일부 지표가 비어 있는 상황이 발생한다.
- 사용자는 누락된 데이터를 수동으로 입력한다.
- 시스템은 수동 입력 기준으로 동일한 검증 및 판정 흐름을 수행한다.
- 자동 수집 실패 여부와 별개로 최종 리포트는 정상 생성되어야 한다.

### 시나리오 C. 입력 오류 발생
- 사용자가 필수 필드를 누락하거나 허용되지 않은 값을 입력한다.
- 시스템은 판정을 수행하지 않는다.
- 시스템은 어떤 필드가 잘못되었는지 명확한 오류 메시지를 반환한다.
- 사용자는 값을 수정한 뒤 다시 실행할 수 있어야 한다.

### 시나리오 D. 일관성 확인
- 사용자는 같은 날 동일한 입력값으로 결과를 다시 확인할 수 있다.
- 시스템은 동일한 입력에 대해 동일한 판정 결과와 동일한 핵심 근거를 반환해야 한다.
- 이를 통해 사용자는 감정이 아닌 규칙 기반 결과를 신뢰할 수 있어야 한다.

## 5. 사용자 기대 결과
- 사용자는 현재 시장이 `추가 조정 필요`, `저점 근접`, `진 바닥 확인` 중 어디에 해당하는지 빠르게 파악할 수 있어야 한다.
- 사용자는 결과뿐 아니라 왜 그런 판정이 나왔는지도 함께 이해할 수 있어야 한다.
- 사용자는 매일 같은 구조의 리포트를 받아 비교 관찰할 수 있어야 한다.

## 6. 시스템 응답 원칙
- 결과는 한국어로 간결하고 전문적으로 제시해야 한다.
- 숫자 데이터와 해석 의견은 분리해야 한다.
- 과도한 확신 표현을 피해야 한다.
- 투자 자문처럼 단정적으로 표현하지 않아야 한다.
- 규칙 기반 판정 결과를 임의 해석으로 덮어쓰지 않아야 한다.
