# MVP 실행 계획

## 1. 문서 목적
이 문서는 빠른 MVP 개발을 전제로, 이 프로젝트를 `단일 Python 텔레그램 봇 애플리케이션`으로 구현하기 위한 실행 계획을 정의한다.

초기 MVP에서는 프론트엔드와 별도 백엔드 서버를 두지 않는다. 핵심 범위는 `수동 입력 -> 검증 -> 판정 -> 리포트 생성 -> 텔레그램 전송`이다.

## 2. MVP 범위
- 포함
  - 수동 입력 기반 실행 경로
  - 입력 검증
  - 규칙 기반 저점 판정
  - 한국어 리포트 생성
  - 텔레그램 메시지 전송
  - 핵심 단위 테스트

- 제외
  - 웹 프론트엔드
  - 별도 API 서버
  - 복잡한 DB 설계
  - 자동 데이터 수집
  - 운영 대시보드

## 3. 핵심 실행 순서
- 1단계: 입력 계약과 설정 경계 확정
- 2단계: 판정 엔진과 리포트 생성 구현
- 3단계: 텔레그램 실행 흐름 연결
- 4단계: 최소 저장 또는 로그 전략 정리
- 5단계: 테스트와 운영 보강

## 4. MVP Task
- [ ] `MVP-01` 입력 데이터 모델을 정의한다.
  - 완료 조건: `06_data_schema.md`의 필드를 반영한 단일 입력 모델이 정의되어 있다.
  - 의존성: `06_data_schema.md`

- [ ] `MVP-02` 입력 검증 계층을 구현한다.
  - 완료 조건: 필수값, 타입, enum 허용값, 하락률 음수 전제를 검증하고 오류를 명확히 반환한다.
  - 의존성: `MVP-01`

- [ ] `MVP-03` 판정 상태 및 도메인 enum을 정의한다.
  - 완료 조건: `추가 조정 필요`, `저점 근접`, `진 바닥 확인`과 입력 enum 값이 코드 상수 또는 enum으로 관리된다.

- [ ] `MVP-04` 저점 판정 규칙 엔진을 구현한다.
  - 완료 조건: 동일 입력에 대해 동일 결과가 나오고, 최종 상태가 3단계 중 하나로 반환된다.
  - 의존성: `MVP-02`, `MVP-03`, `04_logic_spec.md`

- [ ] `MVP-05` 판정 근거 구조를 정의한다.
  - 완료 조건: 상태뿐 아니라 어떤 규칙이 반영됐는지 추적 가능한 결과 구조가 반환된다.
  - 의존성: `MVP-04`

- [ ] `MVP-06` 리포트 생성기를 구현한다.

  - 의존성: `MVP-05`, `05_prompt_spec.md`

- [ ] `MVP-07` 투자 자문 방지 문체 가드를 적용한다.
  - 완료 조건: 과도한 확신 표현, 규칙 엔진 무시, 근거 없는 업종 추천이 출력되지 않는다.
  - 의존성: `MVP-06`, `05_prompt_spec.md`

- [ ] `MVP-08` 설정 및 환경변수 계층을 구현한다.
  - 완료 조건: 텔레그램 토큰, 채널 ID, 실행 모드가 `.env` 또는 환경변수로 분리된다.

- [ ] `MVP-09` 단일 실행 유스케이스를 구현한다.
  - 완료 조건: `입력 -> 검증 -> 판정 -> 리포트 -> 전송`이 하나의 애플리케이션 흐름으로 동작한다.
  - 의존성: `MVP-02`, `MVP-05`, `MVP-06`, `MVP-08`

  - 의존성: `MVP-08`, `MVP-09`

- [ ] `MVP-11` 수동 입력 진입점을 구현한다.
  - 완료 조건: CLI 또는 텔레그램 명령을 통해 수동 입력 데이터로 전체 흐름을 실행할 수 있다.
  - 의존성: `MVP-09`, `MVP-10`

- [ ] `MVP-12` 최소 저장 또는 로그 전략을 정의한다.

- [ ] `MVP-13` 오류 처리 및 운영 로그를 구현한다.

- [ ] `MVP-14` 단위 테스트를 구축한다.

  - 의존성: `MVP-09`, `MVP-10`, `MVP-11`, `MVP-14`
