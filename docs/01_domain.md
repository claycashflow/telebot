# 시장 저점 판독 봇 도메인 정의

## 1. 문제 정의
한국 주식시장(KOSPI/KOSDAQ)은 변동성이 큰 구간에서 "지금이 바닥에 가까운지", "추가 하락 가능성이 더 큰지"를 일관되게 판단하기 어렵다.

개인 투자자는 장 마감 후 여러 지표를 확인하지만, 다음과 같은 문제가 반복된다.

- 하락률, 이격도, 변동성, 매크로 변수 등을 종합 판단하기 어렵다.
- 같은 데이터를 보더라도 그날그날 해석이 달라져 일관성이 떨어진다.
- 감정적 대응으로 인해 과도한 낙관 또는 과도한 비관에 치우치기 쉽다.

이 프로젝트는 장 마감 기준 데이터를 바탕으로 시장 상태를 규칙 기반으로 판독하고, 그 근거를 한국어 리포트로 전달하는 봇이 필요한 도메인을 정의한다.

## 2. 왜 이 문제가 중요한가
- 시장 바닥 여부에 대한 해석이 흔들리면 사용자는 같은 데이터에도 매일 다른 판단을 내리게 된다.
- 변동성이 커질수록 감정적 대응 가능성이 높아지고, 이는 과도한 낙관 또는 과도한 비관으로 이어질 수 있다.
- 사용자는 단순 시황 요약이 아니라, 저점 여부를 일관된 기준으로 해석하는 보조 도구를 필요로 한다.

## 3. 도메인의 핵심 질문
이 도메인은 아래 질문에 답하는 문제로 정의한다.

- 현재 시장은 아직 `추가 조정 필요` 구간인가
- 시장은 `저점 근접` 상태로 볼 수 있는가
- 시장은 `진 바닥 확인` 단계로 판단 가능한가

즉, 이 도메인은 단순 시황 요약이 아니라 저점 여부를 일관된 기준으로 판독하는 문제를 다룬다.

## 4. 대상 사용자
- 한국 주식시장 마감 후 시장 상태를 빠르게 점검하려는 개인 투자자
- 감정이 아니라 규칙 기반으로 시장을 해석하고 싶은 사용자
- KOSPI/KOSDAQ의 저점 신호를 매일 같은 형식으로 보고 싶은 사용자

## 5. 문제 범위
- 문제 범위는 한국 주식시장(KOSPI/KOSDAQ) 장 마감 이후의 시장 상태 판독이다.
- 판단 대상은 개별 종목이 아니라 시장 전체의 저점 상태다.
- 입력 데이터는 하락률, 이격도, 변동성, 이동평균선 지지, 바닥 패턴, 매크로 변수, 이익 사이클 등 저점 판독과 직접 관련된 지표로 한정한다.
- 결과는 `추가 조정 필요`, `저점 근접`, `진 바닥 확인` 중 하나의 상태 판독이어야 한다.

## 6. 제외 범위
- 종목 추천 또는 매수/매도 신호 자동 생성
- 실시간 장중 판단
- 뉴스 감성 분석 기반 판정
- 복잡한 예측 모델 또는 머신러닝 기반 매매 전략

## 7. 문서 간 관계
- 이 문서는 문제와 범위를 정의하는 최상위 도메인 문서다.
- PRD 문서는 이 도메인 정의를 바탕으로 제품이 이 문제를 어떻게 풀고 언제 사용되는지 구체화한다.
- 사용자 시나리오 문서는 PRD를 보조하며 실제 사용 흐름과 예외 흐름을 보여준다.
- 로직 문서는 저점 판정 규칙과 점수 체계를 정의한다.
- 프롬프트 문서는 사용자에게 보여줄 리포트의 서술 형식을 정의한다.
- 데이터 스키마 문서는 입력 데이터의 구조와 필드 의미를 정의한다.
