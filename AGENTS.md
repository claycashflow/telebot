# AGENTS.md

## 목적
- 이 저장소는 한국 주식시장(KOSPI/KOSDAQ) 장 마감 데이터를 바탕으로 시장 저점 여부를 판독하는 텔레그램 봇 프로젝트다.
- 시스템의 1차 목적은 시장 상태를 아래 3단계 중 하나로 일관되게 분류하는 것이다.
- 분류 단계: `추가 조정 필요`, `저점 근접`, `진 바닥 확인`

## 작업 원칙
- 모든 구현과 답변은 `docs/01_domain.md`, `docs/02_prd_check_bottom_bot.md`, `docs/03_user_scenario.md`, `docs/04_logic_spec.md`, `docs/05_prompt_spec.md`, `docs/06_data_schema.md`를 우선 기준으로 삼는다.
- 규칙 기반 판정 결과를 최우선으로 사용한다.
- 문서에 명시된 판정 로직을 임의 해석으로 덮어쓰지 않는다.
- 불확실한 항목은 추정으로 채우지 말고, 누락 데이터 또는 가정임을 명시한다.
- 이 프로젝트의 결과는 투자 판단 보조용이며, 투자 자문처럼 단정적으로 표현하지 않는다.

## 도메인 규칙
- 하락률, 이격도, 변동성, 이동평균선 지지, 바닥 패턴, 유가, 미국 GDP, 반도체 이익 전망을 핵심 신호로 본다.
- 점수 계산 또는 상태 판정 로직을 수정할 때는 반드시 `docs/04_logic_spec.md`와 일치해야 한다.
- `disparity_20`, `disparity_60`, `vkospi`, `ma50_support`, `ma60_support`, `bottom_pattern`, `semiconductor_earnings_view` 등의 필드 의미를 바꾸지 않는다.
- `bottom_pattern` 허용값은 `V_attempt`, `W_forming`, `W_second_bottom`, `Panic_capitulation`으로 유지한다.
- `semiconductor_earnings_view` 허용값은 `positive`, `neutral`, `negative`로 유지한다.

## 출력 원칙
- 사용자 대상 출력은 기본적으로 한국어로 작성한다.
- 리포트는 가능하면 아래 순서를 유지한다.
- `현 시점 저점 판독 결과`
- `판단 근거 (데이터 분석)`
- `매크로 상황 분석`
- `최종 투자 의견`
- `주목할 업종`
- 문체는 간결하지만 전문적으로 유지한다.
- 숫자 데이터와 해석 의견은 분리해서 제시한다.
- 과도한 확신 표현, 근거 없는 업종 추천, 규칙 엔진을 무시한 서술을 금지한다.

## 구현 원칙
- Python 3.11 기준을 전제로 작업한다.
- 설정값과 민감 정보는 `.env`를 사용한다.
- 장애 상황에서도 수동 입력으로 동작 가능한 구조를 유지한다.
- 자동 데이터 수집 기능을 추가하더라도 수동 입력 경로를 제거하지 않는다.
- 테스트 가능한 로직은 함수 단위로 분리하고 테스트 코드를 포함한다.
- 데이터 스키마를 변경할 때는 입력 예시, 검증 로직, 리포트 생성부를 함께 점검한다.

## 데이터 취급
- 입력 데이터 스키마는 `docs/06_data_schema.md`를 기준으로 유지한다.
- 날짜, 지수 종가, 하락률, 이격도, 변동성, 매크로 지표, 업종 전망 필드는 명시적인 타입과 의미를 보존한다.
- 하락률은 최근 고점 대비 하락률이며 음수 값이라는 전제를 유지한다.
- 스키마에 새 필드를 추가할 때는 필드 의미, 허용값, 리포트 반영 방식까지 함께 문서화한다.

## 변경 시 체크리스트
- 변경이 PRD 목표와 직접 연결되는가
- 판정 단계 3종과 로직 명세를 훼손하지 않는가
- 한국어 출력과 리포트 형식을 유지하는가
- 투자 자문처럼 보일 수 있는 표현을 피했는가
- 테스트 또는 검증 경로가 있는가
